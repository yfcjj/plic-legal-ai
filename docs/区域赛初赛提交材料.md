### 2. 项目需求分析

#### 2.1 痛点问题

**传统起诉状准备的三大痛点:**

1. **信息提取效率低**
   - 律师需要从传统起诉状、合同、聊天记录等非结构化文本中手动提取关键要素
   - 重复劳动多,容易遗漏重要信息
   - 不同案件类型需要提取的字段差异大

2. **文书格式不规范**
   - 最高院2022年推行要素式起诉状,但很多律师不熟悉格式
   - 表格字段多(100+字段),手动填写易出错
   - 各地法院对格式要求有细微差异

3. **法律风险难识别**
   - 物业费纠纷常涉及诉讼时效、违约金过高、催缴义务等风险
   - 金融借款纠纷涉及担保方式、利率合规、复利计算等复杂问题
   - 传统方式依赖律师经验,新手律师容易踩坑

#### 2.2 目标用户

**主要用户群体:**
1. **律师事务所** - 提高文书制作效率,降低执业风险
2. **物业公司** - 批量处理物业费纠纷案件
3. **金融机构** - 银行、小贷公司处理借款合同纠纷
4. **法律援助中心** - 为公众提供免费法律服务

**用户画像:**
- 张律师(35岁,执业5年): 每月处理20-30起物业费纠纷,希望提高效率
- 李法务(28岁,公司法务): 负责银行不良资产处置,需要快速生成起诉状
- 王主任(45岁,法律援助): 资源有限,需要AI工具辅助服务更多群众

### 3. 解决方案思路

#### 3.1 核心设计理念

**"AI辅助 + 规则引擎 + 风险预警"三位一体**

1. **AI智能提取** - 利用大模型从非结构化文本中提取结构化要素
2. **规则引擎兜底** - 用正则表达式处理标准格式文本,确保准确性
3. **风险预警系统** - 内置法律规则库,自动识别常见法律风险

### 4. 关键功能说明

#### 4.1 核心功能模块

**功能一:智能案情分析**
- 输入:自然语言案情描述
- 输出:被告识别、管辖法院建议、要素抽取
- 技术实现:调用后端AI接口,返回结构化数据

**功能二:规则提取引擎**
- 支持:从标准格式起诉状中提取要素
- 特点:快速准确,无需AI消耗
- 适用场景:格式规范的文书

**功能三:AI智能提取**
- 支持:处理复杂格式、非标准文本
- 特点:理解能力强,可处理歧义
- 适用场景:手写文书、聊天记录等

**功能四:风险预警系统**
- 物业案件:违约金风险、诉讼时效、催缴义务等
- 金融案件:担保方式、利率合规、复利计算等
- 实现:内置规则库,自动扫描字段

**功能五:要素式起诉状生成**
- 符合最高院2022年格式要求
- 支持100+字段的完整表单
- 自动打印优化,横线对齐、字体统一

#### 4.2 技术亮点

**1. 分离式架构设计**

### 6. AI工具使用说明(工作流与prompt)

#### 6.1 AI提取工作流

**Step 1: 用户输入案情文本**
```

被告:张卫国,男,1973年4月28日出生,汉族

...

```

**Step 2: 后端调用OpenAI API**
```python
# FastAPI后端代码
@app.post("/api/extract")
async def extract_elements(request: ExtractRequest):
    prompt = f"""
    你是一个专业的法律文书分析助手。请从以下{text}中提取结构化要素。
    
    案件类型:{case_type}
    
    请提取以下字段:
    - 原告名称、住所地、法定代表人、联系电话
    - 被告姓名、性别、出生日期、住址、电话
    - 诉讼请求(物业费金额、违约金、标的总额)
    - 事实与理由(合同信息、欠费计算、催缴情况)
    
    返回JSON格式。
    """
    
    response = await openai.ChatCompletion.acreate(
        model="gpt-4",
        messages=[{"role": "user", "content": prompt}],
        temperature=0
    )
    
    return {"success": True, "data": response.choices[0].message.content}
```

### 2. 技术难点与解决方案

#### 难点一:正则表达式匹配准确率

**挑战:** 不同律师撰写的起诉状格式差异大,正则表达式难以覆盖所有情况

**解决方案:**
1. 多轮迭代优化正则规则
2. 添加容错机制(如`[：:]`匹配中英文冒号)
3. 混合AI兜底,正则失败时调用AI

```javascript
// 容错示例
const pm = text.match(/原告[：:]\s*([^\n]+)/);
```

---

#### 难点二:打印样式一致性

**挑战:** 
- 不同浏览器打印样式不一致
- 输入框宽度未继承导致横线太短
- 空值时横线消失

**解决方案:**
```javascript
// 继承原输入框宽度
const inlineWidth = input.style.width;
const computedWidth = window.getComputedStyle(input).width;
const finalWidth = inlineWidth || computedWidth || '100px';

span.style.width = finalWidth;
span.style.minWidth = finalWidth;

// 空值保留占位
span.textContent = value || '\u00A0';
```

---

#### 难点三:Vue插值表达式在打印时不渲染

**挑战:** 签名区使用`{{ prop.courtName }}`,打印时显示原始插值文本

**解决方案:**
```javascript
// 打印前替换插值表达式
text = text.replace(/\{\{\s*prop\.(\w+)\s*\}\}/g, (_, fieldName) => {
    const value = prop[fieldName];
    return value == null || value === '' ? ' ' : String(value);
});
```

    const risks = [];

    const penalty = Math.max(
        cleanAmount(prop.penaltyAmount), 
        cleanAmount(prop.penaltyAmount2)
    );
    if (penalty > 0) {
        risks.push({
            title: '违约金风险',
            content: '司法实践中,物业服务合同纠纷的违约金诉求...'
        });
    }
    
    // 诉讼时效风险
    if (isOverThreeYears(prop.feeEndDate)) {
        risks.push({
            title: '诉讼时效风险',
            content: '物业费适用诉讼时效(通常3年)...'
        });
    }
    
    return risks;
};
