# PLIC 部署指南

## 方案一:GitHub Pages(推荐,免费)

### 步骤:

1. **创建GitHub仓库**
   ```bash
   cd /d/Code/PLIC
   git init
   git add index.html
   git commit -m "初始提交"
   ```

2. **在GitHub上创建新仓库**
   - 访问 https://github.com/new
   - 仓库名: `plic-legal-ai`
   - 设为Public(公开)
   - 不要初始化README

3. **推送代码**
   ```bash
   git remote add origin https://github.com/你的用户名/plic-legal-ai.git
   git branch -M main
   git push -u origin main
   ```

4. **启用GitHub Pages**
   - 进入仓库Settings → Pages
   - Source选择: Deploy from a branch
   - Branch选择: main / root
   - 点击Save

5. **等待部署**
   - 约1-2分钟后,访问: `https://你的用户名.github.io/plic-legal-ai/`

---

## 方案二:Vercel(最快,免费)

### 步骤:

1. **访问Vercel**
   - 打开 https://vercel.com
   - 用GitHub账号登录

2. **导入项目**
   - 点击"New Project"
   - 选择你的GitHub仓库
   - 点击"Import"

3. **配置项目**
   - Framework Preset: Other
   - Root Directory: `./`
   - 点击"Deploy"

4. **获取链接**
   - 部署完成后,你会得到一个类似:
   - `https://plic-legal-ai.vercel.app`

---

## 方案三:Netlify(拖拽部署,最简单)

### 步骤:

1. **访问Netlify**
   - 打开 https://netlify.com
   - 注册账号

2. **拖拽部署**
   - 将 `index.html` 文件拖入Netlify页面
   - 等待上传完成(几秒钟)

3. **获取链接**
   - 会自动生成一个链接,例如:
   - `https://amazing-pudding-123456.netlify.app`

---

## 方案四:本地局域网分享(临时测试)

### 步骤:

1. **使用Python启动本地服务器**
   ```bash
   cd /d/Code/PLIC
   python -m http.server 8080
   ```

2. **查看本机IP**
   ```bash
   ipconfig
   # 找到IPv4地址,例如: 192.168.1.100
   ```

3. **分享链接**
   - 局域网内其他设备访问:
   - `http://192.168.1.100:8080/index.html`

**注意:** 此方法只能在同一WiFi/局域网内访问

## 常见问题

### Q1: 部署后页面空白?
检查文件名是否为 `index.html`(小写)

### Q2: GitHub Pages一直显示部署中?
- 等待1-2分钟
- 检查是否选择了正确的分支(main)

### Q3: Vercel部署失败?
- 确保仓库是Public
- 检查根目录设置是否为 `./`

### Q4: 需要配置域名吗?
不需要,默认域名已经足够好用
