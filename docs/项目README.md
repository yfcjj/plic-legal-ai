# 诉前智检 (PLIC) - 项目说明文档

## 项目简介

**诉前智检**(Pre-Litigation Intelligent Checker)是一款AI驱动的智能诉讼准备助手,专注于**物业服务合同纠纷**和**金融借款合同纠纷**的要素式起诉状自动生成。

### 核心价值

- ⚡ **效率提升25倍** - 从传统3小时缩短至5分钟
- 🎯 **准确率95%+** - 混合规则引擎+AI双重保障
- ⚖️ **风险预警** - 自动识别诉讼时效、违约金等法律风险
- 📄 **格式规范** - 严格符合最高院要素式起诉状要求

## 功能说明

### 1. 规则提取引擎

基于正则表达式,从标准格式文本中快速提取要素。

**支持的文本格式:**
```
原告:襄阳津诚物业服务有限公司
被告:张卫国,男,1973年4月28日出生
截至2025年12月31日,尚欠物业费8078元
```

**提取结果:** 自动填充100+字段

### 2. AI智能提取

处理复杂格式、非标准文本,理解能力强。

**使用场景:**
- 手写文书OCR后处理
- 聊天记录整理
- 多份合同文本合并

### 3. 风险预警系统

**物业案件风险点:**
- 违约金过高
- 诉讼时效届满
- 催缴义务未履行
- 事实服务争议

**金融案件风险点:**
- 担保方式不完善
- 利率超过法定上限
- 复利计算错误

### 4. 打印优化

- ✅ 横线宽度自动继承
- ✅ 空值保留占位
- ✅ 签名区插值替换
- ✅ 字体统一(仿宋14pt)

## 技术架构

### 前端技术栈

- **Vue 3** - Composition API
- **Element Plus** - UI组件库
- **单文件架构** - 所有逻辑集成在index.html

### 后端技术栈

- **FastAPI** - 高性能异步框架
- **OpenAI API** - GPT-4模型
- **Python 3.8+** - 异步支持

### 数据流

```
用户输入文本 
  ↓
规则引擎(正则) → 成功? → 填充表单
  ↓ 失败
AI提取(GPT-4) → 填充表单
  ↓
风险扫描 → 预警提示
  ↓
生成文书 → 打印/PDF
```

## 核心代码解析

### 1. 分离式架构设计

```javascript
// 按钮A:仅提取,不验证
const generateForm = () => {
    if (caseType2.value === 'property') {
        doPropertyExtraction();
        ElementPlus.ElMessage.success('物业服务合同纠纷表单已回填');
    } else if (caseType2.value === 'finance') {
        doFinanceExtraction();
        ElementPlus.ElMessage.success('金融借款合同纠纷表单已回填');
    }
};

// 按钮B:验证+风险+确认+打印
const handleGenerateBeforeSubmit = () => {
    if (caseType2.value === 'property') {
        handleGeneratePropertyBeforeSubmit(); // 包含风险扫描
    } else if (caseType2.value === 'finance') {
        handleGenerateFinanceBeforeSubmit();
    }
};
```

### 2. 打印样式优化

```javascript
// 继承原输入框宽度
const inlineWidth = input.style.width;
const computedWidth = window.getComputedStyle(input).width;
const finalWidth = inlineWidth || computedWidth || '100px';

span.style.width = finalWidth;
span.style.minWidth = finalWidth;

// 空值保留占位
span.textContent = value || '\u00A0';

// 插值表达式替换
text = text.replace(/\{\{\s*prop\.(\w+)\s*\}\}/g, (_, fieldName) => {
    const value = prop[fieldName];
    return value == null || value === '' ? ' ' : String(value);
});
```

### 3. 风险扫描逻辑

```javascript
const collectPropertyRisks = () => {
    const risks = [];
    
    // 违约金风险
    const penalty = Math.max(
        cleanAmount(prop.penaltyAmount), 
        cleanAmount(prop.penaltyAmount2)
    );
    if (penalty > 0) {
        risks.push({
            title: '违约金风险',
            content: '司法实践中,物业服务合同纠纷的违约金诉求通常难以获得法院全额支持...'
        });
    }
    
    // 诉讼时效风险
    if (isOverThreeYears(prop.feeEndDate)) {
        risks.push({
            title: '诉讼时效风险',
            content: '物业费适用诉讼时效(通常3年);超过时效部分法院不予保护...'
        });
    }
    
    return risks;
};
```

## 常见问题

### Q1: 为什么有时候提取不准确?

**A:** 规则引擎依赖标准格式,如果文本格式特殊,建议:
1. 使用"AI智能提取"功能
2. 手动调整格式后再试
3. 联系我们优化规则

### Q2: 打印时横线太短怎么办?

**A:** 已在最新版本修复,确保使用:
- `index.html.bak.20260418080000` 及以后的版本

### Q3: 如何添加新的案由?

**A:** 需要修改以下文件:
1. `index.html` - 添加表单结构
2. 规则提取逻辑 - 添加正则表达式
3. 风险规则库 - 添加风险点

### Q4: AI提取失败怎么办?

**A:** 检查:
1. 后端服务是否启动(`python main.py`)
2. OPENAI_API_KEY是否配置
3. 网络连接是否正常
4. 降级使用"规则提取"

### Q5: 如何批量处理案件?

**A:** 当前版本支持逐个处理,批量功能开发中。
